Problem · 发现问题 Problem · Identifying the Pain

为什么要做这个工具？ Why Build This Tool?

在日常工作中，内部培训活动的台账管理完全依赖人工Excel记录，存在效率低、易出错、难查询等核心痛点。我决定借助数字化工具自主搭建一套半自动化管理系统来解决这些问题。 In daily work, internal training records were managed entirely through manual Excel entries — inefficient, error-prone, and hard to query. I decided to leverage digital tools to independently build a semi-automated management system to address these pain points.

📋
手工记录效率低 Manual Recording Inefficient
每次活动后需手动整理Excel，字段不统一，耗时且容易遗漏关键信息。 Manual Excel sorting after each event, with inconsistent fields — time-consuming and prone to omissions.
🔍
查询统计繁琐 Tedious Queries & Stats
要统计某人的参与记录或某类活动的数据，需要人工筛选，效率极低。 Counting one person's participation or filtering by activity type required manual sorting — extremely slow.
📊
缺乏数据洞察 Lack of Data Insight
无法直观看到培训频次、参与趋势等数据，难以支撑管理决策。 No clear visibility into training frequency or participation trends — hard to support management decisions.

核心录入 + Excel导出 Core Input + Excel Export

第一版聚焦解决最核心的痛点：让记录变得简单，让导出变得一键完成。用最小可用产品验证工具的可行性与实际价值。 The first version focused on the core pain: simplify recording, enable one-click export. Used MVP to validate the tool's feasibility and real value.

  • 培训内容、日期、学时、参与人员录入Input fields: content, date, duration, participants
  • 实时展示所有培训记录列表Real-time list of all training records
  • 一键导出为标准Excel格式One-click export to standard Excel format
V2
功能增强
解决新需求 Enhancement
New Use Cases

个人记录查询 + 定向导出 Personal Query + Targeted Export

V1上线后发现新需求：需要快速查询某位成员的个人参与记录并单独导出。V2针对性地增加了个人维度的查询与导出功能。 After V1 launched, a new need emerged: quickly query an individual's records and export separately. V2 added personal-dimension query and export functions accordingly.

  • 按姓名查找个人培训记录Search personal training records by name
  • 个人记录单独导出为ExcelExport individual records to Excel separately
  • 优化整体布局与交互体验Improved layout and interaction experience
V3
完整系统
数据驱动 Full System
Data-Driven

数据看板 + 完整数据管理 Dashboard + Full Data Management

V3是一次质的跃升——从工具升级为系统。新增可视化数据看板，让管理者一眼掌握培训全貌；完善数据管理能力，支持备份、恢复与操作日志；同步规划知识库助手方案，探索权限隔离架构。 V3 marks a qualitative leap — from tool to system. Added visualization dashboard so managers can grasp the full picture at a glance; improved data management with backup, restore, and operation logs; concurrently designed a knowledge assistant solution exploring permission isolation architecture.

  • 数据看板：总次数、总时长、参与人次实时统计Dashboard: real-time stats on sessions, duration, participants
  • 可视化图表：培训类型分布 + 月度参与趋势Charts: training type distribution + monthly trends
  • 多维筛选：按类型、年份、月份、关键词搜索Multi-dimensional filters: by type, year, month, keyword
  • 数据管理:JSON备份与恢复、操作日志追踪Data management: JSON backup/restore, operation log tracking
  • 知识库助手方案：规划前后端分离、权限隔离架构Knowledge assistant: front-end / back-end separation, permission isolation

这个项目让我真正理解了什么是产品思维——不是有了技术才去找问题，而是发现真实痛点之后，再用最合适的工具去解决它。每一次迭代都来自真实使用反馈，而不是凭空想象"用户可能需要什么"。这种"识别痛点 → 快速验证 → 持续优化"的方法论，同样适用于在大型企业成熟系统基础上做局部体验优化与工具推广。 This project gave me a real understanding of product thinking — not finding problems because you have the technology, but using the most fitting tool after spotting a real pain. Every iteration came from real usage feedback, not from imagining "what users might need". This methodology of "spot pain → rapid validate → continuously optimize" applies equally well to local experience optimization and tool adoption within mature enterprise systems.
